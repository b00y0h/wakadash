# Technology Stack

**Project:** wakafetch Homebrew Distribution
**Researched:** 2026-02-13
**Confidence:** HIGH

## Recommended Stack

### Core Technologies

| Technology | Version | Purpose | Why Recommended |
|------------|---------|---------|-----------------|
| GoReleaser | v2.13.3+ (use `~> v2`) | Complete release automation - builds, packages, signs, publishes | Industry standard for Go CLI releases. Handles multi-platform builds, Homebrew tap generation, checksums, signing, and GitHub releases in one tool. Active development with 15,557+ GitHub stars. Latest release January 10, 2026. |
| Go | 1.24+ (stable recommended) | Build toolchain | Required by GoReleaser. Go 1.24+ is minimum. Use `stable` in GitHub Actions for latest stable release. Go 1.26 released February 2026. |
| GitHub Actions | v4+ (checkout), v5+ (setup-go), v6.4.0+ (goreleaser-action) | CI/CD automation | Native GitHub integration, free for public repos, well-maintained actions. GoReleaser Action v6.4.0 released August 2025. |
| Homebrew Tap | Repository structure following `homebrew-*` naming | Binary distribution for macOS/Linux | Standard package manager for macOS developers. GoReleaser generates and updates formula automatically. Use repository name like `homebrew-wakafetch` or `homebrew-tap`. |

### Security & Verification

| Technology | Version | Purpose | Why Recommended |
|------------|---------|---------|-----------------|
| SHA-256 checksums | Built-in GoReleaser | Artifact verification | Default checksum algorithm in GoReleaser. Automatically generated for all release artifacts. Users can verify with `sha256sum -c checksums.txt`. |
| Cosign (optional) | Latest | Keyless signing of checksums | Modern alternative to GPG. Supports GitHub OIDC for keyless signing. Optional unless enterprise distribution required. |
| GPG signing (optional) | Built-in GoReleaser | Traditional artifact signing | Legacy option for checksum signing. Only needed if users require GPG verification. |

### Authentication Tokens

| Token Type | Scope | Purpose | When to Use |
|------------|-------|---------|-------------|
| `GITHUB_TOKEN` | Single repository | Release to same repo as source code | Default for basic releases. Auto-generated by GitHub Actions. Use for creating releases in source repo. |
| Personal Access Token (PAT) | Cross-repository | Publish to separate Homebrew tap repo | Required when tap is separate repository. Needs `repo` permissions. Store as `HOMEBREW_TAP_TOKEN` secret. |
| GitHub App Token (recommended alternative) | Fine-grained, ephemeral | Modern replacement for PAT | More secure than PAT. Short-lived tokens with minimal blast radius. Use `actions/create-github-app-token` action. |

### macOS Notarization (Optional)

| Tool | Purpose | When Required |
|------|---------|---------------|
| gon | Sign and notarize macOS binaries | Optional unless targeting Mac App Store or enterprise distribution. Requires Apple Developer Account ($99/year). |
| GoReleaser notarize | Built-in notarization support | Alternative to gon with no external dependencies. Works on any OS. |
| Apple Developer Certificate | Code signing | Required for notarization. Certificate type: "Developer ID Application". |

## Installation

```bash
# Local development - install GoReleaser
# Via Homebrew (macOS/Linux)
brew install goreleaser/tap/goreleaser

# Via Go install
go install github.com/goreleaser/goreleaser/v2@latest

# Verify installation
goreleaser --version

# Initialize configuration
goreleaser init

# Test release locally (no publishing)
goreleaser release --snapshot --clean

# Validate configuration
goreleaser check
```

## GitHub Actions Workflow Setup

```yaml
name: Release

on:
  push:
    tags:
      - "*"

permissions:
  contents: write  # Required for creating releases
  packages: write  # If pushing Docker images to GHCR
  issues: write    # If closing milestones

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Required for changelog generation

      - name: Setup Go
        uses: actions/setup-go@v5
        with:
          go-version: stable

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v6
        with:
          version: "~> v2"
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          # For separate Homebrew tap repository:
          HOMEBREW_TAP_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}
```

## Alternatives Considered

| Category | Recommended | Alternative | Why Not Alternative |
|----------|-------------|-------------|---------------------|
| Release automation | GoReleaser | Manual scripts | Error-prone, time-consuming, lacks multi-platform support, no built-in Homebrew integration |
| Release automation | GoReleaser | semantic-release | Primarily for NPM packages, lacks native Homebrew/Go toolchain integration |
| CI/CD | GitHub Actions | GitLab CI / CircleCI | GitHub Actions is native to GitHub, free for public repos, tight GoReleaser integration |
| Homebrew tap | Separate repository | Core Homebrew | Third-party taps required for non-official software. Core Homebrew only for widely-used tools |
| Token type | GitHub App Token | PAT (classic) | PATs are long-lived, broad scope, higher security risk. GitHub steering toward fine-grained tokens |
| Signing | Cosign (keyless) | GPG | GPG requires key management, manual setup. Cosign with OIDC is simpler for GitHub Actions |
| macOS signing | Skip unless needed | gon / notarization | Requires $99/year Apple Developer Account. Only needed for App Store or enterprise. CLI taps work unsigned |

## What NOT to Use

| Avoid | Why | Use Instead |
|-------|-----|-------------|
| PAT (classic) tokens | Long-lived, broad permissions, security risk. GitHub deprecating in favor of fine-grained tokens | GitHub App tokens via `actions/create-github-app-token` or fine-grained PATs |
| Manual release scripts | Error-prone, hard to maintain, lacks checksums/signing | GoReleaser automated workflow |
| Homebrew formula in source repo | Violates Homebrew conventions, harder to maintain | Separate `homebrew-*` tap repository |
| Old GitHub Actions versions | Missing features, potential security issues | checkout@v4, setup-go@v5, goreleaser-action@v6+ |
| SHA-1 or MD5 checksums | Cryptographically weak, deprecated | SHA-256 (GoReleaser default) or SHA-512 |
| Building on macOS runners for basic CLI | Expensive, slow, unnecessary for cross-compilation | Use ubuntu-latest with GoReleaser cross-compilation |

## Stack Patterns by Variant

**If releasing to same repository (simple case):**
- Use `GITHUB_TOKEN` only
- Single `.goreleaser.yaml` in source repo
- Homebrew formula in `taps` section of config
- No separate tap repository needed (GoReleaser creates it)

**If using separate Homebrew tap repository (recommended):**
- Create `homebrew-wakafetch` repository first
- Generate PAT or GitHub App token with `repo` permissions
- Add token as `HOMEBREW_TAP_TOKEN` secret
- Configure `taps` section in `.goreleaser.yaml` to point to separate repo

**If building for macOS with notarization:**
- Add Apple Developer Account credentials
- Install gon or use GoReleaser built-in notarization
- Add signing steps before GoReleaser release
- Increases complexity, only use if required by distribution model

**If targeting enterprise distribution:**
- Add Cosign keyless signing with OIDC
- Generate SBOM (Software Bill of Materials)
- Add attestations for supply chain security
- See `goreleaser/example-supply-chain` repository

## Configuration Quick Reference

**Minimal .goreleaser.yaml for Homebrew tap:**

```yaml
version: 2

builds:
  - env: [CGO_ENABLED=0]
    goos: [linux, darwin, windows]
    goarch: [amd64, arm64]

archives:
  - format: tar.gz
    name_template: "{{ .ProjectName }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}"

brews:
  - repository:
      owner: b00y0h
      name: homebrew-wakafetch
      token: "{{ .Env.HOMEBREW_TAP_TOKEN }}"
    directory: Formula
    homepage: "https://github.com/b00y0h/wakafetch"
    description: "WakaTime statistics fetcher CLI"
    license: "MIT"

checksum:
  name_template: "checksums.txt"
  algorithm: sha256

changelog:
  sort: asc
```
