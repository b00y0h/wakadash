Update GetTheme() to:
1. Add case-insensitive matching (handles "Dracula", "DRACULA", "dracula")
2. Log a warning for non-empty invalid theme names (helps users debug typos)
3. Continue returning Dracula as fallback (no breaking change)

Replace the GetTheme function:
```go
import (
    "log"
    "strings"

    "github.com/charmbracelet/lipgloss"
)

// GetTheme returns a Theme by name (case-insensitive).
// Returns the Dracula theme if the name is not recognized.
// Logs a warning for non-empty invalid names to help users debug config typos.
func GetTheme(name string) Theme {
    normalizedName := strings.ToLower(strings.TrimSpace(name))

    switch normalizedName {
    case "dracula":
        return Dracula
    case "nord":
        return Nord
    case "gruvbox":
        return Gruvbox
    case "monokai":
        return Monokai
    case "solarized":
        return Solarized
    case "tokyonight":
        return TokyoNight
    default:
        // Log warning for non-empty invalid names (empty = first run, expected)
        if normalizedName != "" {
            log.Printf("Warning: unknown theme %q, using 'dracula' instead. Available: %v",
                name, strings.Join(AllThemes(), ", "))
        }
        return Dracula
    }
}
```

This makes theme lookup:
- Case-insensitive (user can write "Dracula" or "dracula" in config)
- Trim whitespace (handles " dracula " with extra spaces)
- Verbose on error (logs available options for typos like "draculaa")
